title: CLI Reference
description: All command-line commands, subcommands, and modes available in the gamut binary.
---

The `gamut` binary is both the daemon and the client. All interaction happens by running it with
different commands.

## Synopsis

```
gamut [OPTIONS] [COMMAND]
```

## Top-level commands

| Command | Description |
|---|---|
| `toggle` or `--toggle` | Toggle the launcher open/closed (default) |
| `daemon` or `--daemon` | Start the background daemon process |
| `quit` or `--quit` | Ask the running daemon to exit |
| `config` | Manage appearance, layout, system options, and shortcuts |
| `-v`, `--version` | Print the version and exit |
| `-h`, `--help` | Print help text and exit |

## Modes

### `--toggle` (default)

Sends a toggle command to the running daemon. If the launcher is currently hidden, it opens it on
the active monitor. If it is open, it closes it.

If no daemon is running, Gamut will start one automatically and then send the toggle command.

```sh
gamut
# same as
gamut --toggle
```

### `--daemon`

Starts the background daemon. The daemon:

- Indexes installed desktop applications
- Listens for IPC commands (toggle, quit)
- Manages the system tray icon
- Handles launcher window lifecycle

Run this once per session (or use [autostart](./getting-started#autostart)):

```sh
gamut --daemon
```

Only one daemon instance runs at a time. Starting a second instance exits immediately.

### `config`

Config is now fully CLI-based. Use it to inspect settings, update individual values, reset values,
and manage shortcuts either directly or through a guided interactive prompt.

```sh
gamut config show
gamut config set appearance.theme dark
gamut config set layout.size large
gamut config shortcut set move-up Ctrl+K
gamut config shortcut interactive
```

Helpful nested help:

```sh
gamut help config
gamut help config shortcut
```

### `--quit`

Sends a quit signal to the running daemon. The daemon will finish any in-flight work and exit
cleanly.

```sh
gamut --quit
```
